以下の対象に対してコードレビューを実施してください。

対象: $ARGUMENTS（未指定の場合は `git diff --staged` の差分全体）

## レビュー観点
1. **正確性**: ロジックエラー、エッジケース、null/undefined の扱い
2. **セキュリティ**: 入力検証、認可、機密情報のハードコード、SQL/コマンドインジェクション
3. **パフォーマンス**: N+1 クエリ、O(n²) 以上の探索、不要な再計算
4. **可読性**: 命名、関数長、責務分離、コメントの過不足
5. **テスト**: 新規ロジックに対するテストの有無

## 重要度区分
- **Critical**: 本番で動作不能/セキュリティ事故レベル。マージ前の修正必須
- **High**: 動作はするが想定外動作やパフォーマンス劣化が出る
- **Medium**: 改善余地あり、別 PR に切り出してもよい
- **Low**: スタイル・命名の提案、任意

## 出力形式
重要度順に「ファイル:行番号 → 問題 → 推奨修正」の3点セットで列挙してください。
最後に Critical/High の合計件数を1行サマリで報告してください。
