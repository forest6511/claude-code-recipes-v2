# レシピ01: 効果的なCLAUDE.mdの書き方

WHY（なぜそうするのか）、WHAT（何をするのか）、HOW（どう実行するのか）の3層構造で `CLAUDE.md` を書き、200 行以下（おおむね 300 行を超えたら分割）に抑える実例です。

## ファイル一覧

- `CLAUDE.md`: Code Style セクション（全体）と Security セクション（抜粋）の最小構成サンプル

## 使い方

自分のプロジェクトのルートに `CLAUDE.md` をコピーし、Why / What / How の3層を保ちながら中身を自プロジェクトのルールに書き換えます。

```bash
cp CLAUDE.md /path/to/your-project/CLAUDE.md
```

## カスタマイズ指針

- **Why**: ルールの背景と意図を散文で書く。エッジケースで Claude Code が判断材料にできる情報を残す
- **What**: 機械的に検証可能な箇条書きで書く。「綺麗に」ではなく「2 スペースインデント」のように具体化する
- **How**: `npm run typecheck` のような実行コマンドに落とし込む
- **行数**: `wc -l CLAUDE.md` で 200 行を目安、300 行を上限として維持する。超える場合は `@import` 記法または `.claude/rules/` に分割する
