# レシピ62: プロンプトチェーンでワークフロー制御

サブエージェントはネスト（入れ子）ができないため、メイン会話がオーケストレーターとなり、サブエージェントを順次呼び出すチェーン方式でワークフローを構築します。各ステップの間に「中間ゲート」を設けることで、方向修正やフィードバックを挟めます。

## ファイル一覧

| ファイル | 説明 |
|---------|------|
| `prompts/chain-workflow.txt` | 4ステップのチェーンワークフロー（分析→計画→実装→検証）のプロンプト例 |
| `prompts/conditional-gate.txt` | テスト結果に応じた条件分岐ゲートのプロンプト例 |
| `prompts/resume-pattern.txt` | セッション中断・再開パターンのプロンプト例 |

## 使い方

```bash
# チェーンワークフローの実行
> 以下のワークフローをステップバイステップで実行してください。
> 各ステップの結果を確認してから次に進んでください。
>
> Step 1（分析）:
>   Exploreサブエージェントで src/api/ のエラーハンドリングの現状を分析する
>
> Step 2（計画）:
>   分析結果を踏まえて、統一エラーハンドリングの設計案を作成する
>   → ここで設計案を私に確認してください
>
> Step 3（実装）:
>   承認された設計案に基づいて実装する
>
> Step 4（検証）:
>   全テストを実行し、既存機能への影響がないことを確認する
```

## チェーン方式の基本構造

```
メイン会話（オーケストレーター）
  |
  +-- Step 1: [Explore] 問題の分析
  |     +-- 分析結果をメイン会話に返す
  |
  +-- 中間ゲート: 分析結果を確認し、計画を承認
  |
  +-- Step 2: [Plan] 解決策の立案
  |     +-- 計画をメイン会話に返す
  |
  +-- 中間ゲート: 計画を確認し、実行を承認
  |
  +-- Step 3: [general-purpose] 実装の実行
  |     +-- 実行結果をメイン会話に返す
  |
  +-- Step 4: [general-purpose] テストと検証
        +-- 最終結果をメイン会話に返す
```

## 中間ゲートの種類

ゲートには大きく 3 種類あります。

- **自動ゲート**: 前ステップの結果を自動的に次ステップに渡す。定型的なワークフローに向く
- **確認ゲート**: ユーザーに結果を提示し承認を求める。重要な判断ポイントに向く
- **条件分岐ゲート**: 結果に応じて次のステップを選択する。エラー処理や分岐ロジックに向く

## 設計のポイント

- **「ここで確認してください」の一文が中間ゲート**: Claude Code はこの指示に従い、結果を提示して承認を待つ
- **条件分岐は明示的な数値ルールで記述する**: 「テスト失敗 3 件以下」「4 件以上」のような閾値で固定する
- **各ステップの結果をファイルに保存する**: `.claude/workflow/step1-result.md` のようなファイルに書き出すことでメイン会話のコンテキスト消費を抑える
- **チェーン設計図を Skills として定義する**: ワークフロー全体を Skills 化し、スラッシュコマンドで再利用可能にする
- **再開可能性を意識する**: `SendMessage` で過去のサブエージェントを再開できる（`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 必要）
